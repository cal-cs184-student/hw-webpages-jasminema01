For each triangle polygon, we first compute the boundary this triangle lives in, then set the x_min, x_max, y_min, y_max respectively based on min/max vertex coordinates. Then for every pixel in this clamped box, we determine if it’s inside a triangle using the helper function point in triangle test with x + 0.5 and y + 0.5 as the sampled pixel center. A point would be inside a triangle if all three edge values have the same sign (either positive or negative to handle counter-clock wise and clockwise vertex orders). If it’s inside according to the test, we fill_pixel(x, y, color).

The algorithm simply uses 3 edge function evaluation (constant per sample), therefore it is no worse than one that checks every sample within a bounding box.

Task 5: "Pixel sampling" for texture mapping

Pixel sampling can be used to determine what color a screen pixel should receive based on a texture image. Each triangle vertex has a texture coordinate (u,v), and for every rasterized pixel inside the triangle, we use barycentric interpolation to compute the pixel’s corresponding (u,v) coordinate. Once we have that, we sample the texture image at that location to determine the pixel’s color. In our implementation to perform texture mapping, after computing the interpolated texture coordinates for each sample point, we applied either neatest or bilinear sampling depending on the selected pixel sampling mode. For the nearest sampling, we chose the texel closest to the (u,v) location. For bilinear sampling, applied linear interpolation on the four neighboring texels to produce a smoother result.

Nearest sampling chooses the texel closest to the (u,v) coordinate and returns its color directly, which is quite simple and fast, as it only requires one texel lookup, but produces more noticeable aliasing when textures are minified. Bilinear sampling looks at the four texels neighboring the (u,v) coordinate and linearly interpolates between them. It first interpolates horizontally between two pairs of textels, then interpolates vertically between those intermediate results. This produces smoother transitions and reduces jaggedness between texels, but it creates a slight bur and is less cost efficient in computation.

The difference between nearest and bilinear sampling is most apparent when texture is magnified, where nearest looks blocky and bilinear looks smooth. When the texture has high-frequency detail, where nearest causes strong aliasing. Overall, bilinear sampling reduces aliasing by smoothing between texels, while nearest sampling keeps the image sharp but intensifies aliasing.

Task 6: "Level Sampling" with mipmaps for texture mapping

Level sampling determines which mipmap resolution to sample from during texture mapping. While pixel sampling (psm) controls how we sample within a mipmap level (nearest or bilinear), level sampling (lsm) controls which mipmap level we use. Instead of sampling only from the full-resolution texture, mipmapping precomputes progressively smaller, prefiltered versions of the texture and selects the level that best matches the screen-space footprint of a pixel. This helps reduce aliasing during minification.

To compute the appropriate mipmap level, we first interpolate the texture coordinates (u,v) at each sub-sample using barycentric weights. We then compute two additional texture coordinates at neighboring screen positions (x+1, y) and (x, y+1), stored in sp.p_dx_uv and sp.p_dy_uv. In Texture::get_level, we estimate the texture footprint by computing the differences in UV coordinates, scaling them by the full-resolution texture width and height to convert to texel space, and taking the maximum magnitude of the footprint in the x and y directions. The mipmap level is computed as:

D = log2(L)

where L represents the texel footprint size. The resulting level is clamped to the valid mipmap range.

In Texture::sample, we use this level according to the selected level sampling mode:

  • L_ZERO: Always sample from mip level 0.
  • L_NEAREST: Round D to the nearest integer mip level.
  • L_LINEAR: Linearly interpolate between the two closest mip levels.
Pixel sampling (nearest or bilinear) is applied independently within the chosen mipmap level, so any combination of psm and lsm works together.

Tradeoffs

Pixel sampling: Nearest sampling is fastest but produces blocky artifacts. Bilinear sampling improves smoothness with a small additional computational cost.

Level sampling (mipmapping): Mipmaps significantly reduce aliasing and moiré patterns during minification by matching texture resolution to pixel footprint. However, they require additional memory to store multiple texture levels and slightly more computation to determine and interpolate between levels.

Increasing samples per pixel (supersampling): Raising the screen-space sampling rate reduces jagged edges and improves overall antialiasing, but it is computationally expensive because shading must be performed multiple times per pixel.

Overall, mipmapping provides a strong balance between visual quality and efficiency, especially for minified textures, while supersampling offers stronger antialiasing at a higher runtime cost.
